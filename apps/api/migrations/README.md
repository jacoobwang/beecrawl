# Database Migrations

BeeCrawl uses `sqlx-cli` migrations. The naming convention mirrors the
timestamp ordering used by `workus-ai`, while preserving SQLx's numeric version
parser:

```text
YYYYMMDDHHMMSS_description.sql
```

The version ID is a UTC timestamp generated by `sqlx migrate add` through
`make migration-new`. It must never be renamed or reused after it has been
applied. SQLx records the version and checksum in `_sqlx_migrations`, so
changing an applied file is rejected.

Use lowercase snake case for `description`:

```bash
mkdir -p ~/.cargo/bin
cargo install sqlx-cli --no-default-features --features postgres,rustls
make migration-new name=add_crawl_webhook
make migrate-up
```

SQLx only accepts digits before the first underscore. For that reason the date
and time are combined in the filename, rather than using workus-ai's visual
`YYYYMMDD_HHMMSS` separator.
